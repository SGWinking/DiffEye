# DiffEye 详细使用教程

本文档说明 DiffEye V0.4 的完整使用流程、参数含义和常见调节方法。

## 1. 基本流程

1. 启动 DiffEye。
2. 打开浏览器访问：

```text
http://localhost:5055
```

3. 放入两张图片：

```text
第一张图：原图、修复前图、基准图
第二张图：生成图、修复后图、待检查图
```

4. 设置参数。
5. 点击“开始比较”。
6. 查看前后对比图、完整结构差异图和右侧聚焦区域。

## 2. 两种比对方式

### 结构比对

适合：

```text
壁画
纹样
图纸
线条
轮廓
修复前后图
AI 生成图和原图
```

结构比对会先提取两张图的线条和边缘，再判断新增、缺失和偏移。

### 像素差异

适合：

```text
截图
渲染图
严格像素一致性检查
```

像素差异更严格，对颜色和细小像素变化也敏感。

## 3. 参数说明

### 灵敏度

控制线条提取的敏感程度。

```text
调高：更容易抓到细线，也更容易误报
调低：只看明显线条，但可能漏掉浅线
```

推荐默认：

```text
5
```

### 最小区域

单位是像素面积。小于这个面积的差异聚集区不会生成聚焦区域。

推荐默认：

```text
2048
```

简单理解：

```text
数值越大，小碎片越容易被过滤，框会更少
数值越小，小变化也会出现，框会更多
```

### 检测数量

最多显示多少个聚焦区域。

推荐默认：

```text
120
```

如果需要显示更多区域，可以调到：

```text
200 或 300
```

### 容错px

允许两张图的线条有多少像素以内的轻微偏移。

推荐默认：

```text
10
```

简单理解：

```text
数值越大：越忽略轻微错位和笔墨边缘变化
数值越小：越严格，小偏移也会被抓出来
```

### 最大区域%

超过整张图这个比例的超大差异聚集区会被过滤。

推荐默认：

```text
60
```

如果图上颜色很多但框很少，可以尝试调高。

### 线条模式

```text
快速 Canny：速度快，适合常规结构审查
精细 DexiNed：AI 边缘检测，适合复杂线条，需要额外模型
```

## 4. 三色结构差异图

完整结构差异图使用三种颜色：

```text
红色：后图新增的线条
蓝色：原图有但后图缺失的线条
黄色：两边都有但位置略有偏移
```

顶部图例可以勾选或取消：

```text
新增
缺失
偏移
```

如果黄色太多，可以先关闭“偏移”，只看新增和缺失。

## 5. 聚焦区域

右侧“聚焦区域”是系统认为差异比较集中的位置。

每个卡片显示：

```text
尺寸
新增数量
缺失数量
偏移数量
密度
```

密度表示该框内差异点占区域面积的比例。密度越高，说明差异越集中。

## 6. 双击局部审查

可以双击：

```text
右侧聚焦区域卡片
完整结构差异图中的编号框
```

弹窗会显示三栏：

```text
原图局部
后图局部
差异局部
```

鼠标滚轮可以放大或缩小弹窗图像。

## 7. 人工标记

弹窗顶部提供人工标记：

```text
通过
复核
问题
忽略
```

当前标记保存在浏览器本地，用于辅助人工审图流程。

## 8. 参数调节建议

### 误报太多

可以尝试：

```text
提高容错px，例如 12 或 15
提高最小区域，例如 3000 或 5000
关闭偏移图层
降低灵敏度
```

### 漏检太多

可以尝试：

```text
降低最小区域
提高检测数量
降低容错px
尝试精细 DexiNed 模式
```

### 图上颜色很多，但框很少

可以尝试：

```text
提高最大区域%
提高检测数量
降低最小区域
```

## 9. DexiNed 使用建议

DexiNed 会识别更多细线，但也更容易受到线条深浅差异影响。

建议起步参数：

```text
线条模式：精细 DexiNed
灵敏度：5
最小区域：2048
容错px：10
最大区域%：60
检测数量：120
```

如果 DexiNed 下红色新增太多，可以提高容错px，或先使用快速 Canny 做常规结构审查。

## 10. 网站部署提示

DiffEye 当前主要面向本地部署。如果要做公开网站或付费服务，需要额外增加：

```text
用户系统
支付系统
任务队列
对象存储
数据隔离
上传大小限制
自动清理机制
隐私政策
AGPL-3.0 对应源码入口
```

如果公开提供网络服务，请遵守 AGPL-3.0，并向用户提供对应源代码。
